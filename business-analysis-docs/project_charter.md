## 🔍 What is a Project Charter?
A project charter is a **short document that officially defines the project**. It answers 3 simple questions:
- What are we building?
- Who is building it?
- What are the boundaries?

Think of it as the "ID card" of the project.

## 2. 🎯 What is the Purpose of This Project?

In simple words:

> "Too many employees are leaving the company. We want to find out **why** they are leaving, **who** is likely to leave next, and **what** the company should do about it."

This project uses **data** to answer all 3 questions.

---

## 3. 🏁 Project Goals (Objectives)

By the end of this project, we will have:

- ✅ Cleaned and analyzed HR data using **SQL and Python**
- ✅ Found the **top reasons** why employees leave
- ✅ Built an **ML model** that predicts which employees are at risk
- ✅ Created a **Power BI dashboard** that HR can use daily
- ✅ Written **recommendations** for how to reduce attrition

---

## 4. 📦 Scope — What We Did and Did Not Do

### ✅ We Did:
- Analyzed 1,470 employee records
- Wrote SQL queries to find attrition patterns
- Built Python notebooks for EDA and ML modeling
- Designed a 5-page Power BI dashboard
- Wrote business recommendations based on findings

### ❌ We Did Not:
- Connect to any real company database
- Build a mobile application
- Work with real-time employee data
- Automate HR processes

## 6. ⚠️ Risks and Challenges

| Risk | What Could Go Wrong | How We Handled It |
|------|--------------------|--------------------|
| Data quality | CSV had missing or wrong values | Cleaned using Python (pandas) |
| Model accuracy | ML model might be wrong sometimes | Used XGBoost + tuned hyperparameters |
| Dashboard performance | Too many visuals can slow down Power BI | Kept only essential visuals per page |

## 7. 📈 Expected Business Impact

If the company acts on this project's recommendations:

- 💰 Save up to **$40,000 per employee** retained
- 📉 Reduce attrition from **16.1% → 11%** (target)
- ⏱️ HR spends less time reacting to resignations, more time on retention
